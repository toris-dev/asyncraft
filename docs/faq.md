# FAQ

## Where should I ask a question?

- **How-to / usage**: GitHub Issue if behavior-focused; otherwise GitHub Discussions.
- **Bug reports**: GitHub Issue with the bug report template.
- **Feature ideas**: GitHub Issue with the feature request template.
- **Security issues**: [SECURITY.md](../SECURITY.md).

## How should I report a bug?

A high-quality bug report should include:

- asyncraft version and Node version
- minimal reproducible snippet
- expected and actual behavior
- environment details (`OS`, package manager)
- logs/stack trace if available

## How do you prioritize issues?

1. Reproducibility
2. API impact
3. Reliability impact
4. Fix effort and testability

## Why was my ticket closed?

Tickets can be closed when:

- duplicate exists
- insufficient reproduction details
- out of project scope
- security vulnerability report belongs in private advisory channel

## How to contribute

1. Review [CONTRIBUTING.md](../CONTRIBUTING.md).
2. Confirm your change is scoped.
3. Open or comment on an issue before large PRs.
4. Include tests for behavior changes.
